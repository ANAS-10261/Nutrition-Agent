# 🥗 NutriBot — AI Nutrition Agent
### Powered by **IBM Watsonx.ai** · Granite Models

A full-stack, AI-powered nutrition assistant built with **Python Flask** and **IBM Watsonx.ai** (Granite 13B). Features a modern responsive chat UI, personalized meal planning, BMI calculator, food analyzer, and family profile support — with deep Indian cuisine and Ayurvedic integration.

---

## ✨ Features

| Feature | Description |
|---|---|
| 💬 **AI Chat** | Conversational nutrition assistant via IBM Granite |
| 📊 **Dashboard** | Personalized BMI, TDEE, macros & hydration stats |
| 🗓️ **Meal Planner** | 1–7 day Indian meal plans generated by AI |
| ⚖️ **BMI Calculator** | Full BMR/TDEE calculation with calorie targets |
| 👨‍👩‍👧‍👦 **Family Profiles** | Unified meal plans for up to 8 family members |
| 🔍 **Food Analyzer** | Instant nutritional breakdown of any dish |
| 🌙 **Dark Mode** | Fully implemented light/dark theme toggle |
| 📱 **Mobile Responsive** | Works on all screen sizes |

---

## 🛠️ Tech Stack

- **Backend**: Python 3.10+, Flask 3.x, IBM Watsonx.ai SDK
- **AI Model**: `ibm/granite-13b-instruct-v2`
- **Frontend**: Bootstrap 5.3, Vanilla JS, Bootstrap Icons
- **Security**: python-dotenv for API key management

## ⚡ Quick Start

### 1. Prerequisites

```bash
# Requires Python 3.10+
python --version

# Install dependencies
pip install -r requirements.txt
```

### 2. Configure Environment

```bash
# Copy the example env file
cp .env.example .env
```

Edit `.env` with your IBM credentials:

```env
IBM_API_KEY=your_ibm_cloud_api_key_here
IBM_PROJECT_ID=your_watsonx_project_id_here
IBM_URL=https://us-south.ml.cloud.ibm.com
FLASK_SECRET_KEY=your_random_secret_key
```

### 3. Get IBM Credentials

| Credential | Where to get it |
|---|---|
| `IBM_API_KEY` | [IBM Cloud IAM → API Keys](https://cloud.ibm.com/iam/apikeys) |
| `IBM_PROJECT_ID` | [IBM watsonx.ai Studio](https://dataplatform.cloud.ibm.com) → Your Project → Manage → General → Project ID |
| `IBM_URL` | Use `https://us-south.ml.cloud.ibm.com` for US South (default) |

### 4. Run the Application

```bash
python app.py
```

Open `http://localhost:5000` in your browser.

## 🤖 Customizing Agent Behavior

All agent customization is done in **one place** — the `AGENT_INSTRUCTIONS` dictionary at the top of [`app.py`](app.py):

```python
AGENT_INSTRUCTIONS = {
    "AGENT_NAME": "NutriBot",
    "AGENT_TONE": "friendly",          # friendly | professional | motivational | clinical
    "DIET_SPECIALIZATION": "indian_regional",
    "LANGUAGE_STYLE": "simple",        # simple | detailed | scientific
    "INCLUDE_INDIAN_FOOD": True,       # Always suggest Indian dishes first
    "RESPONSE_LANGUAGE": "english",    # english | hinglish | hindi
    "MAX_FAMILY_MEMBERS": 8,

    "SAFETY_RULES": [
        "Never prescribe medication or replace a licensed dietitian.",
        # Add / remove rules as needed
    ],

    "CALORIE_DISCLAIMER": "Your custom disclaimer text...",

    "PERSONALITY": "Describe your agent's personality and expertise...",
}
```

### Customization Examples

**For a Keto-specialized agent:**
```python
"DIET_SPECIALIZATION": "keto",
"PERSONALITY": "You are a strict keto diet specialist...",
```

**For a children's nutrition agent:**
```python
"AGENT_NAME": "KidNutri",
"AGENT_TONE": "friendly",
"SAFETY_RULES": [..., "Always recommend child-safe portion sizes."],
```

**For a clinical/hospital context:**
```python
"AGENT_TONE": "clinical",
"LANGUAGE_STYLE": "scientific",
"INCLUDE_INDIAN_FOOD": False,
```

## 🌐 API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Main application |
| `POST` | `/api/chat` | AI chat with Granite |

## 🚀 Deployment

### Option 1 — Local (Development)

```bash
python app.py
```

### Option 2 — Gunicorn (Production)

```bash
gunicorn -w 4 -b 0.0.0.0:5000 app:app
```

### Option 3 — Docker

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
EXPOSE 5000
CMD ["gunicorn", "-w", "4", "-b", "0.0.0.0:5000", "app:app"]
```

```bash
docker build -t nutribot .
docker run -p 5000:5000 --env-file .env nutribot
```

### Option 4 — IBM Cloud Code Engine

```bash
# Install IBM Cloud CLI + Code Engine plugin
ibmcloud login --apikey $IBM_API_KEY
ibmcloud ce project create --name nutribot-project
ibmcloud ce app create \
  --name nutribot \
  --image icr.io/your-namespace/nutribot:latest \
  --env-from-secret nutribot-secrets \
  --port 5000
```

### Option 5 — Heroku

```bash
# Create Procfile
echo "web: gunicorn app:app" > Procfile

heroku create my-nutribot
heroku config:set IBM_API_KEY=xxx IBM_PROJECT_ID=yyy
git push heroku main
```

## 🔒 Security

- **API keys** are loaded from `.env` — never hardcoded
- `.env` is excluded from git via `.gitignore`
- The `/api/agent-config` endpoint only exposes non-sensitive config
- Session secret key should be a cryptographically random string in production:
  ```python
  python -c "import secrets; print(secrets.token_hex(32))"
  ```

## 🐛 Troubleshooting

| Issue | Solution |
|---|---|
| `IBM_API_KEY not set` | Copy `.env.example` → `.env` and fill values |
| `Watsonx.ai init failed` | Verify API key at [cloud.ibm.com/iam/apikeys](https://cloud.ibm.com/iam/apikeys) |
| `Model not found` | Ensure your IBM region URL matches `IBM_URL` |
| `Demo mode active` | App runs without AI if credentials missing — add `.env` to fix |
| Port conflict | Change `PORT=5001` in `.env` |
